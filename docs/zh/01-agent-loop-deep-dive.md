# Agent 循环深度解析

> 本文深入分析 AI Agent 的核心循环机制，帮助后端开发者理解其工作原理。

## 一、循环的本质

### 1.1 为什么需要循环？

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     单次调用 vs 循环调用                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  单次调用（传统 API）：                                                 │
│  ┌─────────┐      ┌─────────┐      ┌─────────┐                         │
│  │  请求   │ ───▶ │   LLM   │ ───▶ │  响应   │                         │
│  └─────────┘      └─────────┘      └─────────┘                         │
│                                                                         │
│  问题：LLM 只能"说话"，不能"做事"                                      │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  循环调用（Agent）：                                                    │
│  ┌─────────┐      ┌─────────┐      ┌─────────┐      ┌─────────┐        │
│  │  请求   │ ───▶ │   LLM   │ ───▶ │ 工具调用│ ───▶ │  执行   │        │
│  └─────────┘      └─────────┘      └─────────┘      └────┬────┘        │
│       ▲                                                     │           │
│       │              ┌─────────┐                            │           │
│       └──────────────│  结果   │◀───────────────────────────┘           │
│                      └─────────┘                                        │
│                                                                         │
│  关键：LLM 决定何时需要使用工具，执行后继续思考                         │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 循环的状态机视角

```
                        Agent Loop 状态机
                        =================

                              ┌───────────────┐
                              │               │
                              │    START      │
                              │               │
                              └───────┬───────┘
                                      │
                                      ▼
                         ┌────────────────────────┐
                         │                        │
                         │     CALL_LLM           │
                         │     调用大模型         │
                         │                        │
                         └────────────┬───────────┘
                                      │
                                      ▼
                         ┌────────────────────────┐
                         │                        │
              ┌──────────│   CHECK_STOP_REASON    │──────────┐
              │          │                        │          │
              │          └────────────────────────┘          │
              │                        │                     │
        end_turn                 tool_use                    │
              │                        │                     │
              ▼                        ▼                     │
    ┌─────────────────┐    ┌────────────────────┐           │
    │                 │    │                    │           │
    │   END           │    │   EXECUTE_TOOLS    │           │
    │   返回文本结果   │    │   执行工具调用     │           │
    │                 │    │                    │           │
    └─────────────────┘    └─────────┬──────────┘           │
                                      │                      │
                                      ▼                      │
                         ┌────────────────────────┐          │
                         │                        │          │
                         │   APPEND_RESULTS       │          │
                         │   追加结果到消息       │          │
                         │                        │          │
                         └────────────┬───────────┘          │
                                      │                      │
                                      └──────────────────────┘
                                             (循环继续)
```

## 三、工具系统详解

### 3.1 工具定义结构

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        工具定义结构                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  工具定义 (JSON Schema):                                                │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ {                                                               │   │
│  │   "name": "edit_file",        // 工具名称                      │   │
│  │   "description": "替换文件中的文本", // 功能描述                │   │
│  │   "input_schema": {            // 输入参数定义                  │   │
│  │     "type": "object",                                          │   │
│  │     "properties": {                                            │   │
│  │       "path": {                                                │   │
│  │         "type": "string",                                      │   │
│  │         "description": "文件路径"                              │   │
│  │       },                                                       │   │
│  │       "old_text": {                                            │   │
│  │         "type": "string",                                      │   │
│  │         "description": "要替换的文本"                          │   │
│  │       },                                                       │   │
│  │       "new_text": {                                            │   │
│  │         "type": "string",                                      │   │
│  │         "description": "新文本"                                │   │
│  │       }                                                        │   │
│  │     },                                                         │   │
│  │     "required": ["path", "old_text", "new_text"]               │   │
│  │   }                                                            │   │
│  │ }                                                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 工具分发机制

```
                        工具分发流程
                        ============

    ┌──────────────────────────────────────────────────────────────────────┐
    │                                                                      │
    │     LLM Response                                                    │
    │         │                                                           │
    │         ▼                                                           │
    │  ┌──────────────────┐                                               │
    │  │  解析 tool_use   │                                               │
    │  │  block.name      │                                               │
    │  │  block.input     │                                               │
    │  └────────┬─────────┘                                               │
    │           │                                                          │
    │           ▼                                                          │
    │  ┌──────────────────────────────────────────────────────────────┐   │
    │  │                    Dispatch Map                               │   │
    │  │                                                               │   │
    │  │  TOOL_HANDLERS = {                                           │   │
    │  │      "bash":       ─────▶  run_bash(**input)                 │   │
    │  │      "read_file":  ─────▶  run_read(**input)                 │   │
    │  │      "write_file": ─────▶  run_write(**input)                │   │
    │  │      "edit_file":  ─────▶  run_edit(**input)                 │   │
    │  │      "task":       ─────▶  run_task(**input)                 │   │
    │  │  }                                                           │   │
    │  │                                                               │   │
    │  └──────────────────────────────────────────────────────────────┘   │
    │           │                                                          │
    │           ▼                                                          │
    │  ┌──────────────────┐                                               │
    │  │   执行 handler   │                                               │
    │  │   返回 result    │                                               │
    │  └──────────────────┘                                               │
    │                                                                      │
    └──────────────────────────────────────────────────────────────────────┘


    Go 后端类比:

    ┌──────────────────────────────────────────────────────────────────────┐
    │                                                                      │
    │  // 类似 http.ServeMux 的路由分发                                   │
    │  handlers := map[string]HandlerFunc{                                │
    │      "bash":       runBash,                                         │
    │      "read_file":  runRead,                                         │
    │      "write_file": runWrite,                                        │
    │  }                                                                  │
    │                                                                      │
    │  handler := handlers[toolName]                                      │
    │  result := handler(input)                                           │
    │                                                                      │
    └──────────────────────────────────────────────────────────────────────┘
```

### 3.3 工具扩展示例

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    如何添加新工具                                        │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  步骤1: 定义工具函数                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ def run_database_query(sql: str) -> str:                        │   │
│  │     """执行数据库查询"""                                         │   │
│  │     try:                                                        │   │
│  │         result = db.execute(sql)                                │   │
│  │         return json.dumps(result)                               │   │
│  │     except Exception as e:                                      │   │
│  │         return f"Error: {e}"                                    │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  步骤2: 添加工具定义                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ TOOLS.append({                                                  │   │
│  │     "name": "db_query",                                         │   │
│  │     "description": "执行 SQL 查询",                             │   │
│  │     "input_schema": {                                           │   │
│  │         "type": "object",                                       │   │
│  │         "properties": {                                         │   │
│  │             "sql": {"type": "string"}                           │   │
│  │         },                                                      │   │
│  │         "required": ["sql"]                                     │   │
│  │     }                                                           │   │
│  │ })                                                              │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  步骤3: 注册到分发器                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ TOOL_HANDLERS["db_query"] = lambda **kw: \                      │   │
│  │     run_database_query(kw["sql"])                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  完成！Agent 现在可以调用 db_query 工具了                              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 四、Stop Reason 机制

### 4.1 Stop Reason 类型

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      Stop Reason 类型                                   │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  ┌─────────────────┬─────────────────────────────────────────────────┐ │
│  │  Stop Reason    │  含义                                          │ │
│  ├─────────────────┼─────────────────────────────────────────────────┤ │
│  │                 │                                                 │ │
│  │  end_turn       │  正常结束，返回文本给用户                       │ │
│  │                 │  LLM 认为任务已完成                             │ │
│  │                 │                                                 │ │
│  ├─────────────────┼─────────────────────────────────────────────────┤ │
│  │                 │                                                 │ │
│  │  tool_use       │  需要调用工具                                   │ │
│  │                 │  LLM 返回了 tool_use block                      │ │
│  │                 │  执行后需要继续循环                             │ │
│  │                 │                                                 │ │
│  ├─────────────────┼─────────────────────────────────────────────────┤ │
│  │                 │                                                 │ │
│  │  max_tokens     │  达到 token 上限                                │ │
│  │                 │  需要处理截断情况                               │ │
│  │                 │                                                 │ │
│  ├─────────────────┼─────────────────────────────────────────────────┤ │
│  │                 │                                                 │ │
│  │  stop_sequence  │  遇到停止序列                                   │ │
│  │                 │  自定义的停止条件                               │ │
│  │                 │                                                 │ │
│  └─────────────────┴─────────────────────────────────────────────────┘ │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 4.2 决策流程

```
                        Stop Reason 决策流程
                        ====================

                              ┌─────────────┐
                              │  LLM 返回   │
                              │  response   │
                              └──────┬──────┘
                                     │
                                     ▼
                        ┌────────────────────────┐
                        │                        │
                        │  stop_reason ==        │
                        │  "tool_use" ?          │
                        │                        │
                        └────────────┬───────────┘
                                     │
                    ┌────────────────┼────────────────┐
                    │                │                │
                   YES               NO              其他
                    │                │                │
                    ▼                ▼                ▼
           ┌──────────────┐  ┌──────────────┐  ┌──────────────┐
           │              │  │              │  │              │
           │ 执行工具     │  │ 返回文本     │  │ 处理特殊情况 │
           │ 追加结果     │  │ 给用户       │  │ (重试/报错)  │
           │ 继续循环     │  │ 结束循环     │  │              │
           │              │  │              │  │              │
           └──────────────┘  └──────────────┘  └──────────────┘
                    │
                    │
                    └──────▶ 返回顶部继续循环
```

### 5.2 上下文压缩策略

```
                        三层压缩策略
                        ============

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│  Layer 1: 微压缩 (micro_compact)                                        │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  时机: 每轮循环                                                 │   │
│  │  策略: 旧的工具结果替换为占位符                                 │   │
│  │  效果: 保留最近 3 个工具结果的完整内容                          │   │
│  │                                                                 │   │
│  │  Before:                                                       │   │
│  │  tool_result: "50000 characters of file content..."            │   │
│  │                                                                 │   │
│  │  After:                                                        │   │
│  │  tool_result: "[Previous: used read_file]"                     │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Layer 2: 自动压缩 (auto_compact)                                       │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  时机: token 超过阈值 (如 50000)                                │   │
│  │  策略: 保存完整记录到磁盘，用 LLM 生成摘要                      │   │
│  │  效果: messages 被替换为摘要                                    │   │
│  │                                                                 │   │
│  │  1. 保存 messages 到 .transcripts/timestamp.jsonl              │   │
│  │  2. 调用 LLM 生成摘要                                          │   │
│  │  3. messages = [摘要消息]                                      │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  Layer 3: 手动压缩 (compact tool)                                       │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  时机: Agent 主动调用 compact 工具                              │   │
│  │  策略: 与自动压缩相同                                           │   │
│  │  效果: Agent 可以在关键时刻主动压缩                             │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 六、安全与限制

### 6.1 危险命令过滤

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        安全过滤机制                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  def run_bash(command: str) -> str:                                    │
│      # 危险命令黑名单                                                   │
│      dangerous = [                                                      │
│          "rm -rf /",      # 删除根目录                                  │
│          "sudo",          # 提权                                        │
│          "shutdown",      # 关机                                        │
│          "reboot",        # 重启                                        │
│          "> /dev/",       # 写入设备                                    │
│      ]                                                                  │
│                                                                         │
│      if any(d in command for d in dangerous):                          │
│          return "Error: Dangerous command blocked"                      │
│                                                                         │
│      # 执行命令...                                                      │
│                                                                         │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  路径安全检查:                                                          │
│                                                                         │
│  def safe_path(p: str) -> Path:                                        │
│      path = (WORKDIR / p).resolve()                                    │
│      if not path.is_relative_to(WORKDIR):                              │
│          raise ValueError(f"Path escapes workspace: {p}")              │
│      return path                                                        │
│                                                                         │
│  防止: ../../../etc/passwd 等路径穿越攻击                              │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 6.2 超时保护

```
                        超时保护机制
                        ============

    ┌──────────────────────────────────────────────────────────────────┐
    │                                                                  │
    │  工具执行超时:                                                   │
```

## 八、关键代码解析

### 8.1 s01_agent_loop.py 核心代码

```python
def agent_loop(messages: list):
    while True:
        # 1. 调用 LLM
        response = client.messages.create(
            model=MODEL, 
            system=SYSTEM, 
            messages=messages,
            tools=TOOLS, 
            max_tokens=8000,
        )
        
        # 2. 追加助手消息
        messages.append({"role": "assistant", "content": response.content})
        
        # 3. 检查是否需要调用工具
        if response.stop_reason != "tool_use":
            return  # 不需要调用工具，任务完成
        
        # 4. 执行所有工具调用
        results = []
        for block in response.content:
            if block.type == "tool_use":
                # 执行工具
                output = run_bash(block.input["command"])
                # 收集结果
                results.append({
                    "type": "tool_result", 
                    "tool_use_id": block.id,
                    "content": output,
                })
        
        # 5. 追加工具结果
        messages.append({"role": "user", "content": results})
        
        # 6. 循环继续，LLM 会看到工具结果
```

### 8.2 Go 伪代码实现

```go
// Agent Loop 的 Go 实现
func AgentLoop(messages *[]Message) {
    for {
        // 1. 调用 LLM
        response, err := llmClient.CreateMessage(ctx, &MessageRequest{
            Model:    model,
            System:   systemPrompt,
            Messages: *messages,
            Tools:    tools,
            MaxTokens: 8000,
        })
        if err != nil {
            log.Fatal(err)
        }
        
        // 2. 追加助手消息
        *messages = append(*messages, Message{
            Role:    "assistant",
            Content: response.Content,
        })
        
        // 3. 检查是否需要调用工具
        if response.StopReason != "tool_use" {
            return // 任务完成
        }
        
        // 4. 执行所有工具调用
        var results []ToolResult
        for _, block := range response.Content {
            if block.Type == "tool_use" {
                output := executeTool(block.Name, block.Input)
                results = append(results, ToolResult{
                    Type:       "tool_result",
                    ToolUseID:  block.ID,
                    Content:    output,
                })
            }
        }
        
        // 5. 追加工具结果
        *messages = append(*messages, Message{
            Role:    "user",
            Content: results,
        })
        
        // 6. 循环继续
    }
}

// 工具执行
func executeTool(name string, input map[string]any) string {
    handlers := map[string]ToolHandler{
        "bash":       runBash,
        "read_file":  runRead,
        "write_file": runWrite,
        "edit_file":  runEdit,
    }
    
    handler, ok := handlers[name]
    if !ok {
        return fmt.Sprintf("Unknown tool: %s", name)
    }
    
    return handler(input)
}
```
