# 多智能体协作详解

> 本文解析多个 AI Agent 如何协作完成任务，包括团队组建、通信协议和自治执行。

## 一、为什么需要多智能体协作？

### 1.1 单 Agent 的局限性

```
┌─────────────────────────────────────────────────────────────────────────┐
│                     单 Agent 的局限性                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  复杂任务的挑战:                                                        │
│                                                                         │
│  用户任务: "开发一个完整的用户管理系统"                                 │
│                                                                         │
│  单 Agent 面临的问题:                                                   │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  1. 上下文限制                                                  │   │
│  │     - 需要处理前端、后端、数据库、测试...                       │   │
│  │     - 信息量太大，上下文不够用                                  │   │
│  │                                                                 │   │
│  │  2. 专业领域                                                    │   │
│  │     - 前端开发需要 React/Vue 知识                               │   │
│  │     - 后端开发需要 API 设计知识                                 │   │
│  │     - 单一 Agent 难以精通所有领域                               │   │
│  │                                                                 │   │
│  │  3. 并行效率                                                    │   │
│  │     - 前端和后端可以并行开发                                    │   │
│  │     - 单 Agent 只能串行处理                                     │   │
│  │                                                                 │   │
│  │  4. 故障隔离                                                    │   │
│  │     - 一个子任务失败可能影响整体                                │   │
│  │     - 需要隔离失败，不影响其他部分                              │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```


## 三、消息总线（Message Bus）

### 3.1 JSONL 消息存储

```
┌─────────────────────────────────────────────────────────────────────────┐
│                        JSONL 消息存储                                    │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  消息格式:                                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ {                                                               │   │
│  │   "type": "message",           // 消息类型                     │   │
│  │   "from": "lead",              // 发送者                       │   │
│  │   "content": "请实现登录功能",  // 消息内容                     │   │
│  │   "timestamp": 1709500000      // 时间戳                       │   │
│  │ }                                                               │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  alice.jsonl 示例:                                                      │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ {"type":"message","from":"lead","content":"实现登录","ts":1}     │   │
│  │ {"type":"message","from":"lead","content":"添加验证","ts":2}     │   │
│  │ {"type":"broadcast","from":"lead","content":"暂停工作","ts":3}   │   │
│  │ {"type":"shutdown_request","from":"lead","request_id":"abc","ts":4} │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  特点:                                                                  │
│  ├── 追加写入 (append-only)，高效                                      │
│  ├── 读取后清空 (drain)，保证消息只处理一次                            │
│  └── 文件存储，重启后消息不丢失                                        │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

## 四、团队协议（Team Protocols）

### 4.1 关闭协议

```
                        关闭协议流程
                        ============

    ┌──────────────────────────────────────────────────────────────────────────┐
    │                                                                          │
    │  目的: 优雅地关闭队友，确保工作不丢失                                   │
    │                                                                          │
    │  ┌─────────────────┐                              ┌─────────────────┐   │
    │  │                 │                              │                 │   │
    │  │  主控 Agent     │                              │    alice        │   │
    │  │    (lead)       │                              │   (coder)       │   │
    │  │                 │                              │                 │   │
    │  └────────┬────────┘                              └────────┬────────┘   │
    │           │                                                 │           │
    │           │ shutdown_request                                │           │
    │           │ {request_id: "abc"}                             │           │
    │           │────────────────────────────────────────────────▶│           │
    │           │                                                 │           │
    │           │                                    alice 决定:   │           │
    │           │                                    可以关闭吗?   │           │
    │           │                                                 │           │
    │           │                                    ┌────────────┴───────┐   │
    │           │                                    │                    │   │
    │           │                               可以 │               不可以│   │
    │           │                                    ▼                    ▼   │
    │           │ shutdown_response            approve: true    approve: false│
    │           │ {request_id: "abc",          reason: "工作已完成"           │
    │           │  approve: true}                                          │   │
    │           │◀─────────────────────────────────────────────────         │   │
    │           │                                                 │           │
    │           │                                    状态: shutdown          │
    │           │                                    线程结束                 │   │
    │           │                                                 │           │
    │           ▼                                                 ▼           │
    │  ┌─────────────────┐                              ┌─────────────────┐   │
    │  │ 记录关闭成功    │                              │ 继续工作        │   │
    │  │ 更新 config    │                              │ 下次再试        │   │
    │  └─────────────────┘                              └─────────────────┘   │
    │                                                                          │
    └──────────────────────────────────────────────────────────────────────────┘

    状态机:

    ┌─────────────┐     shutdown_request      ┌─────────────┐
    │             │ ─────────────────────────▶│             │
    │   working   │                           │  pending    │
    │             │◀──────────────────────────│  shutdown   │
    └─────────────┘    shutdown_response      └─────────────┘
           │             (reject)                    │
           │                                        │ shutdown_response
           │                                        │ (approve)
           │                                        ▼
           │                                 ┌─────────────┐
           │                                 │             │
           │                                 │  shutdown   │
           │                                 │  (结束)     │
           │                                 │             │
           │                                 └─────────────┘
           │
           └────────────────────────────────────────────────▶
```

## 五、自治智能体（Autonomous Agents）

### 5.1 自治生命周期

```
                        自治智能体生命周期
                        ==================

    ┌──────────────────────────────────────────────────────────────────────────┐
    │                                                                          │
    │                          ┌─────────────┐                                 │
    │                          │   spawn     │                                 │
    │                          │   启动      │                                 │
    │                          └──────┬──────┘                                 │
    │                                 │                                        │
    │                                 ▼                                        │
    │    ┌───────────────────────────────────────────────────────────────┐    │
    │    │                                                               │    │
    │    │                    WORK PHASE (工作阶段)                      │    │
    │    │                                                               │    │
    │    │    ┌─────────────────────────────────────────────────────┐   │    │
    │    │    │                                                     │   │    │
    │    │    │  while True:                                        │   │    │
    │    │    │      # 检查收件箱                                   │   │    │
    │    │    │      inbox = read_inbox()                           │   │    │
    │    │    │      if inbox:                                      │   │    │
    │    │    │          处理消息                                   │   │    │
    │    │    │                                                     │   │    │
    │    │    │      # 调用 LLM                                     │   │    │
    │    │    │      response = LLM(messages)                       │   │    │
    │    │    │                                                     │   │    │
    │    │    │      # 执行工具                                     │   │    │
    │    │    │      if tool_use:                                   │   │    │
    │    │    │          execute_tools()                            │   │    │
    │    │    │                                                     │   │    │
    │    │    │      # 检查是否空闲                                 │   │    │
    │    │    │      if idle_requested:                             │   │    │
    │    │    │          break  # 进入 IDLE 阶段                    │   │    │
    │    │    │                                                     │   │    │
    │    │    └─────────────────────────────────────────────────────┘   │    │
    │    │                                                               │    │
    │    └───────────────────────────────────────────────────────────────┘    │
    │                                 │                                        │
    │                                 │ idle_requested                         │
    │                                 ▼                                        │
    │    ┌───────────────────────────────────────────────────────────────┐    │
    │    │                                                               │    │
    │    │                    IDLE PHASE (空闲阶段)                      │    │
    │    │                                                               │    │
    │    │    status = "idle"                                            │    │
    │    │                                                               │    │
    │    │    for i in range(IDLE_TIMEOUT / POLL_INTERVAL):             │    │
    │    │        sleep(POLL_INTERVAL)  # 每5秒轮询一次                 │    │
    │    │                                                               │    │
    │    │        # 检查收件箱                                           │    │
    │    │        inbox = read_inbox()                                   │    │
    │    │        if inbox:                                              │    │
    │    │            有新消息 → 返回 WORK PHASE                        │    │
    │    │                                                               │    │
    │    │        # 检查任务看板                                         │    │
    │    │        unclaimed = scan_unclaimed_tasks()                     │    │
    │    │        if unclaimed:                                          │    │
    │    │            自动认领任务 → 返回 WORK PHASE                    │    │
    │    │                                                               │    │
    │    │    # 超时，没有新工作                                         │    │
    │    │    shutdown                                                   │    │
    │    │                                                               │    │
    │    └───────────────────────────────────────────────────────────────┘    │
    │                                 │                                        │
    │                    ┌────────────┴────────────┐                          │
    │                    │                         │                          │
    │               有新工作                   超时无工作                     │
    │                    │                         │                          │
    │                    ▼                         ▼                          │
    │            返回 WORK PHASE            ┌─────────────┐                   │
    │                                          │  shutdown   │                   │
    │                                          │   结束      │                   │
    │                                          └─────────────┘                   │
    │                                                                            │
    └──────────────────────────────────────────────────────────────────────────┘
```

### 5.2 任务看板扫描

```
                        任务看板扫描机制
                        ================

    ┌──────────────────────────────────────────────────────────────────────┐
    │                                                                      │
    │  .tasks/ 目录中的任务:                                               │
    │                                                                      │
    │  task_1.json:                    task_2.json:                        │
    │  ┌─────────────────────────┐    ┌─────────────────────────┐        │
    │  │ {                       │    │ {                       │        │
    │  │   "id": 1,              │    │   "id": 2,              │        │
    │  │   "status": "completed",│    │   "status": "pending",  │        │
    │  │   "owner": "alice"      │    │   "owner": "",          │ ◀── 可认领
    │  │ }                       │    │   "blockedBy": []       │        │
    │  └─────────────────────────┘    │ }                       │        │
    │                                  └─────────────────────────┘        │
    │                                                                      │
    │  task_3.json:                                                        │
    │  ┌─────────────────────────┐                                        │
    │  │ {                       │                                        │
    │  │   "id": 3,              │                                        │
    │  │   "status": "pending",  │                                        │

    │  ┌────────────────────────────────────────────────────────────────┐ │
    │  │                                                                │ │
    │  │  def scan_unclaimed_tasks():                                  │ │
    │  │      unclaimed = []                                           │ │
    │  │      for task_file in .tasks/task_*.json:                     │ │
    │  │          task = load(task_file)                               │ │
    │  │          if (task.status == "pending"                         │ │
    │  │              and not task.owner        # 没有人认领           │ │
    │  │              and not task.blockedBy):  # 没有被阻塞           │ │
    │  │              unclaimed.append(task)                           │ │
    │  │      return unclaimed                                         │ │
    │  │                                                                │ │
    │  └────────────────────────────────────────────────────────────────┘ │
    │                                                                      │
    │  认领过程:                                                           │
    │  ┌────────────────────────────────────────────────────────────────┐ │
    │  │                                                                │ │
    │  │  1. 找到可认领的任务 (task_2)                                  │ │
    │  │  2. 更新 task_2.json:                                         │ │
    │  │     {                                                          │ │
    │  │       "id": 2,                                                 │ │
    │  │       "status": "in_progress",                                 │ │
    │  │       "owner": "alice"         ← 设置所有者                   │ │
    │  │     }                                                          │ │
    │  │  3. 开始执行任务                                               │ │
    │  │                                                                │ │
    │  └────────────────────────────────────────────────────────────────┘ │
    │                                                                      │
    └──────────────────────────────────────────────────────────────────────┘
```

### 7.2 Go 后端实现示例

```go
// 消息总线的 Go 实现
type Message struct {
    Type      string                 `json:"type"`
    From      string                 `json:"from"`
    Content   string                 `json:"content"`
    Timestamp int64                  `json:"timestamp"`
    Extra     map[string]interface{} `json:"extra,omitempty"`
}

type MessageBus struct {
    inboxDir string
    mu       sync.Mutex
}

func (b *MessageBus) Send(sender, to, content, msgType string, extra map[string]interface{}) error {
    msg := Message{
        Type:      msgType,
        From:      sender,
        Content:   content,
        Timestamp: time.Now().Unix(),
        Extra:     extra,
    }
    
    data, err := json.Marshal(msg)
    if err != nil {
        return err
    }
    
    // 追加写入 JSONL 文件
    inboxPath := filepath.Join(b.inboxDir, to+".jsonl")
    f, err := os.OpenFile(inboxPath, os.O_APPEND|os.O_CREATE|os.O_WRONLY, 0644)
    if err != nil {
        return err
    }
    defer f.Close()
    
    _, err = f.WriteString(string(data) + "\n")
    return err
}

func (b *MessageBus) ReadInbox(name string) ([]Message, error) {
    inboxPath := filepath.Join(b.inboxDir, name+".jsonl")
    
    data, err := os.ReadFile(inboxPath)
    if err != nil {
        if os.IsNotExist(err) {
            return nil, nil
        }
        return nil, err
    }
    
    var messages []Message
    for _, line := range bytes.Split(data, []byte("\n")) {
        if len(line) == 0 {
            continue
        }
        var msg Message
        if err := json.Unmarshal(line, &msg); err != nil {
            continue
        }
        messages = append(messages, msg)
    }
    
    // 清空收件箱
    os.WriteFile(inboxPath, []byte{}, 0644)
    
    return messages, nil
}

// 任务看板扫描
func ScanUnclaimedTasks(tasksDir string) ([]Task, error) {
    files, err := filepath.Glob(filepath.Join(tasksDir, "task_*.json"))
    if err != nil {
        return nil, err
    }
    
    var unclaimed []Task
    for _, f := range files {
        data, err := os.ReadFile(f)
        if err != nil {
            continue
        }
        
        var task Task
        if err := json.Unmarshal(data, &task); err != nil {
            continue
        }
        
        if task.Status == "pending" && task.Owner == "" && len(task.BlockedBy) == 0 {
            unclaimed = append(unclaimed, task)
        }
    }
    
    return unclaimed, nil
}
```
