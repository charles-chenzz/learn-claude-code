# 任务规划与进度管理

> 本文解析 AI Agent 如何通过 TodoWrite 和子智能体实现任务规划和进度管理。

## 一、为什么需要任务规划？

### 1.1 问题场景

```
┌─────────────────────────────────────────────────────────────────────────┐
│                    没有 Agent 规划时的问题                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  用户任务: "帮我重构用户认证模块"                                       │
│                                                                         │
│  没有 TodoWrite 的 Agent:                                               │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  Agent: "好的，我来重构..."                                     │   │
│  │  (开始工作)                                                     │   │
│  │  (做了几步后)                                                   │   │
│  │  Agent: "我刚才在做什么来着？"                                  │   │
│  │  (忘记上下文，重复工作或遗漏步骤)                               │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  有 TodoWrite 的 Agent:                                                 │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  Agent: "好的，让我先规划一下任务..."                           │   │
│  │  Todo: [ ] 分析现有代码                                        │   │
│  │        [ ] 设计新的认证流程                                    │   │
│  │        [ ] 实现新的认证逻辑                                    │   │
│  │        [ ] 更新相关测试                                        │   │
│  │        [ ] 验证功能正常                                        │   │
│  │                                                                 │   │
│  │  (每完成一步，更新状态，不会遗漏)                               │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 1.2 核心理念

```
                        核心理念
                        ========

    ┌──────────────────────────────────────────────────────────────────┐
    │                                                                  │
    │   "没有计划的 agent 走哪算哪"                                    │
    │                                                                  │
    │   先列步骤再动手，完成率翻倍                                     │
    │                                                                  │
    │   ┌──────────────────────────────────────────────────────────┐  │
    │   │                                                          │  │
    │   │   传统编程:  代码定义好每一步该怎么做                     │  │
    │   │                                                          │  │
    │   │   AI Agent:  AI自己决定要做什么，但需要跟踪进度           │  │
    │   │                                                          │  │
    │   │   TodoWrite: 让AI能够"记住"自己的计划                     │  │
    │   │                                                          │  │
    │   └──────────────────────────────────────────────────────────┘  │
    │                                                                  │
    └──────────────────────────────────────────────────────────────────┘
```

```
                                     │
                        开始处理     │
                                     │
                                     ▼
                              ┌─────────────┐

                              │             │
                              └──────┬──────┘

                   ┌─────────────────┼─────────────────┐
```

### 2.3 Nag 提醒机制

```
                        Nag 提醒机制
                        ============

    ┌──────────────────────────────────────────────────────────────────┐
    │                                                                  │
    │  问题: Agent 可能忘记更新 Todo                                   │
    │                                                                  │
    │  解决: 如果连续 3 轮没有调用 todo 工具，自动注入提醒             │
    │                                                                  │
    │  ┌────────────────────────────────────────────────────────────┐ │
    │  │                                                            │ │
    │  │  rounds_since_todo = 0                                     │ │
    │  │                                                            │ │
    │  │  while True:                                               │ │
    │  │      ...                                                   │ │
    │  │                                                            │ │
    │  │      if used_todo:                                         │ │
    │  │          rounds_since_todo = 0                             │ │
    │  │      else:                                                 │ │
    │  │          rounds_since_todo += 1                            │ │
    │  │                                                            │ │
    │  │      if rounds_since_todo >= 3:                            │ │
    │  │          # 注入提醒                                        │ │
    │  │          results.insert(0, {                               │ │
    │  │              "type": "text",                               │ │
    │  │              "text": "<reminder>Update your todos.</reminder>" │ │
    │  │          })                                                │ │
    │  │                                                            │ │
    │  └────────────────────────────────────────────────────────────┘ │
    │                                                                  │
    └──────────────────────────────────────────────────────────────────┘

    时间线示例:

    轮次 1: Agent 调用 todo 更新    → rounds_since_todo = 0
    轮次 2: Agent 调用 read_file    → rounds_since_todo = 1
    轮次 3: Agent 调用 edit_file    → rounds_since_todo = 2
    轮次 4: Agent 调用 bash         → rounds_since_todo = 3 → 触发提醒!
                                     ┌─────────────────────────────────┐
                                     │ <reminder>Update your todos.</reminder>
                                     └─────────────────────────────────┘
```

## 三、子智能体（Subagent）机制

### 3.1 为什么需要子智能体？

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      上下文污染问题                                      │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  主 Agent 执行探索任务时的问题:                                         │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  用户: "分析整个项目的代码结构"                                 │   │
│  │                                                                 │   │
│  │  主 Agent 开始工作:                                            │   │
│  │  - 读取 file1.py (2000行) → messages 增长                      │   │
│  │  - 读取 file2.py (3000行) → messages 继续增长                  │   │
│  │  - 读取 file3.py (4000行) → messages 爆炸                      │   │
│  │  - ...                                                         │   │
│  │                                                                 │   │
│  │  问题: 探索过程产生的大量中间结果污染了主对话上下文            │   │
│  │  后果: 主 Agent 的有效上下文被压缩，后续工作质量下降           │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  使用子智能体的解决方案:                                                │
│                                                                         │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │                                                                 │   │
│  │  主 Agent: "我需要分析项目结构，让子智能体来做"                 │   │
│  │                                                                 │   │
│  │  调用 task 工具，启动子智能体                                   │   │
│  │  子智能体有自己的 messages = [] (全新上下文)                    │   │
│  │  子智能体: 大量探索工作...                                      │   │
│  │  子智能体: 返回精简的摘要                                       │   │
│  │                                                                 │   │
│  │  主 Agent 收到: "项目包含以下模块: ..." (简洁摘要)              │   │
│  │  主 Agent 的上下文保持干净                                      │   │
│  │                                                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

### 3.2 子智能体架构图

```
                        子智能体架构
                        ============

    ┌──────────────────────────────────────────────────────────────────────┐
    │                                                                      │
    │  主 Agent (Parent)                    子智能体 (Subagent)            │
    │  ┌──────────────────────┐            ┌──────────────────────┐       │
    │  │                      │            │                      │       │
    │  │  messages = [        │            │  messages = []       │       │
    │  │    {user: "分析项目"},│  dispatch  │      ↓               │       │
    │  │    {assistant: ...}, │ ─────────▶ │  {user: prompt}      │       │
    │  │    ...               │            │      ↓               │       │
    │  │  ]                   │            │  {assistant: ...}    │       │
    │  │                      │            │      ↓               │       │
    │  │  上下文保持干净      │  summary   │  ... (大量工作)       │       │
    │  │                      │ ◀───────── │      ↓               │       │
    │  │  收到: "摘要..."     │            │  return 摘要         │       │
    │  │                      │            │                      │       │
    │  └──────────────────────┘            └──────────────────────┘       │
    │                                                                      │
    │  关键区别:                                                           │
    │  ├── 子智能体有全新的 messages[]                                    │
    │  ├── 子智能体的工具集可能受限 (不能递归创建子智能体)                │
    │  ├── 子智能体工作完成后，只返回摘要给主 Agent                       │
    │  └── 子智能体的上下文被丢弃，不污染主 Agent                         │
    │                                                                      │
    └──────────────────────────────────────────────────────────────────────┘
```

### 3.3 子智能体工作流程

```
                        子智能体调用流程
                        ================

    ┌────────────────────────────────────────────────────────────────────────┐

    │  │ 主 Agent 循环                                                    │ │
    │  │                                                                  │ │
    │  │  LLM 思考: "这是一个探索任务，会产生大量中间结果，               │ │
    │  │           应该用子智能体来做"                                    │ │
    │  │                                                                  │ │
    │  │  LLM 输出:                                                       │ │
    │  │  ┌────────────────────────────────────────────────────────────┐ │ │
    │  │  │ tool_use: task                                             │ │ │
    │  │  │ description: "查找 Redis 使用"                             │ │ │
    │  │  │ prompt: "搜索整个项目，找出所有使用 Redis 的地方，         │ │ │
    │  │  │         列出文件名和具体位置"                              │ │ │
    │  │  └────────────────────────────────────────────────────────────┘ │ │
    │  │                              │                                   │ │
    │  │                              ▼                                   │ │
    │  │  ┌────────────────────────────────────────────────────────────┐ │ │
    │  │  │ 子智能体启动                                               │ │ │
    │  │  │                                                            │ │ │
    │  │  │  子智能体 messages = [                                     │ │ │
    │  │  │    {user: "搜索整个项目，找出所有使用 Redis 的地方..."}   │ │ │
    │  │  │  ]                                                         │ │ │
    │  │  │                                                            │ │ │
    │  │  │  子智能体循环:                                             │ │ │
    │  │  │  ┌──────────────────────────────────────────────────────┐ │ │ │
    │  │  │  │ 轮次 1: bash("grep -r redis .")                       │ │ │ │
    │  │  │  │         → 大量匹配结果                                │ │ │ │
    │  │  │  ├──────────────────────────────────────────────────────┤ │ │ │
    │  │  │  │ 轮次 2: read_file("cache.py")                         │ │ │ │
    │  │  │  │         → 完整文件内容                                │ │ │ │
    │  │  │  ├──────────────────────────────────────────────────────┤ │ │ │
    │  │  │  │ 轮次 3: read_file("queue.py")                         │ │ │ │
    │  │  │  │         → 完整文件内容                                │ │ │ │
    │  │  │  ├──────────────────────────────────────────────────────┤ │ │ │
    │  │  │  │ ... (更多探索)                                        │ │ │ │
    │  │  │  ├──────────────────────────────────────────────────────┤ │ │ │
    │  │  │  │ 轮次 N: 任务完成，生成摘要                            │ │ │ │
    │  │  │  │         stop_reason = "end_turn"                     │ │ │ │
    │  │  │  │         text = "找到以下位置使用 Redis: ..."         │ │ │ │
    │  │  │  └──────────────────────────────────────────────────────┘ │ │ │
    │  │  │                                                            │ │ │
    │  │  │  子智能体结束，返回摘要文本                                │ │ │
    │  │  │  子智能体上下文被丢弃                                      │ │ │
    │  │  │                                                            │ │ │
```

### 5.2 Go 后端实现示例

```go
// Todo 管理器的 Go 实现
type TodoItem struct {
    ID     string `json:"id"`
    Text   string `json:"text"`
    Status string `json:"status"` // pending, in_progress, completed
}

type TodoManager struct {
    items []TodoItem
    mu    sync.RWMutex
}

func (t *TodoManager) Update(items []TodoItem) error {
    t.mu.Lock()
    defer t.mu.Unlock()
    
    // 验证
    if len(items) > 20 {
        return errors.New("max 20 todos allowed")
    }
    
    inProgressCount := 0
    for _, item := range items {
        if item.Status == "in_progress" {
            inProgressCount++
        }
    }
    if inProgressCount > 1 {
        return errors.New("only one task can be in_progress")
    }
    
    t.items = items
    return nil
}

func (t *TodoManager) Render() string {
    var lines []string
    completed := 0
    
    for _, item := range t.items {
        var marker string
        switch item.Status {
        case "pending":
            marker = "[ ]"
        case "in_progress":
            marker = "[>]"
        case "completed":
            marker = "[x]"
            completed++
        }
        lines = append(lines, fmt.Sprintf("%s #%s: %s", marker, item.ID, item.Text))
    }
    
    lines = append(lines, fmt.Sprintf("\n(%d/%d completed)", completed, len(t.items)))
    return strings.Join(lines, "\n")
}

// 子智能体调用
func RunSubagent(prompt string) string {
    // 创建新的消息上下文
    subMessages := []Message{
        {Role: "user", Content: prompt},
    }
    
    // 子智能体循环
    for i := 0; i < 30; i++ {
        response := callLLM(subMessages, childTools)
        subMessages = append(subMessages, Message{
            Role:    "assistant",
            Content: response.Content,
        })
        
        if response.StopReason != "tool_use" {
            break
        }
        
        results := executeTools(response.Content)
        subMessages = append(subMessages, Message{
            Role:    "user",
            Content: results,
        })
    }
    
    // 只返回最后的文本摘要
    return extractTextFromResponse(response)
}
```
