## 一、什么是 AI Agent？（核心概念）

### 1.1 最简单的理解

```
┌─────────────────────────────────────────────────────────────────┐
│                      AI Agent 本质                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│    传统程序：你写代码 → 程序执行固定逻辑 → 返回结果              │
│                                                                 │
│    AI Agent：你给目标 → AI自己思考 → 调用工具 → 完成任务        │
│                                                                 │
│    关键区别：AI Agent 有"自主决策"能力                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 类比理解

```
┌──────────────────────────────────────────────────────────────────┐
│                    餐厅类比                                      │
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  传统程序 = 自动贩卖机                                           │
│  ├── 你按按钮A                                                   │
│  └── 机器吐出商品A（固定流程）                                   │
│                                                                  │
│  AI Agent = 智能服务员                                           │
│  ├── 你说"我饿了"                                               │
│  ├── 服务员思考：这位客人想吃什么？                              │
│  ├── 服务员行动：查看菜单、询问口味、下单                        │
│  └── 服务员交付：端上符合你需求的餐点                            │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 2.2 伪代码表示（给Go开发者的直觉理解）

```go
// Agent Loop 的 Go 伪代码
func AgentLoop(messages []Message) {
    for {
        // 1. 调用大模型
        response := LLM.Call(messages)
        
        // 2. 追加助手回复到历史
        messages.Append(response)
        
        // 3. 判断是否需要调用工具
        if response.StopReason != "tool_use" {
            return // 任务完成，退出循环
        }
        
        // 4. 执行所有工具调用
        results := []ToolResult{}
        for _, block := range response.Content {
            if block.Type == "tool_use" {
                // 执行工具并获取结果
                output := ExecuteTool(block.Name, block.Input)
                results = append(results, ToolResult{
                    ToolUseID: block.ID,
                    Content:   output,
                })
            }
        }
        
        // 5. 将工具结果追加到消息历史
        messages.Append(Message{
            Role:    "user",
            Content: results,
        })
        
        // 6. 继续循环，让LLM看到工具结果后继续思考
    }
}
```

│                    │                                                    │

│                                                                         │
│  Python Agent 代码:              Go 后端对应写法:                       │
│                                                                         │
│  ┌─────────────────────────┐    ┌─────────────────────────────────┐    │
│  │ TOOL_HANDLERS = {       │    │ var handlers = map[string]      │    │
│  │     "bash": run_bash,   │    │     HandlerFunc{                │    │
│  │     "read": run_read,   │    │     "bash":  runBash,           │    │
│  │     "write": run_write, │    │     "read":  runRead,           │    │
│  │ }                       │    │     "write": runWrite,          │    │
│  │                         │    │ }                               │    │
│  │ def exec_tool(name,     │    │                                 │    │
│  │              args):     │    │ func execTool(name string,      │    │
│  │     handler =           │    │            args map[string]any) │    │
│  │       TOOL_HANDLERS     │    │            string {             │    │
│  │         [name]          │    │     h := handlers[name]         │    │

│  │       handler(**args)   │    │ }                               │    │
│  └─────────────────────────┘    └─────────────────────────────────┘    │
│                                                                         │
│  核心思想一致：通过注册表解耦调用者和执行者                              │
